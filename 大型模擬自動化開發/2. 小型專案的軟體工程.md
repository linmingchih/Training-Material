# 軟體工程


軟體工程 (Software Engineering) 是一門綜合性的學科，它將工程學的方法、技術和原則應用於軟體的設計、開發、測試、部署與維護。其核心目標是在預算內、按時交付高質量、可靠且可維護的軟體產品。它不僅僅是編寫程式碼，更涵蓋了從需求分析到產品生命週期管理的整個過程。

在傳統的觀念中，軟體工程常被認為是大型複雜專案的專屬，因為這些專案通常涉及龐大的團隊、漫長的開發週期和嚴格的規範。然而，即使是「小型專案」，引入軟體工程的概念和實踐也同樣重要且能帶來巨大的效益。


### 軟體工程的關鍵要素 (適用於小型專案)

-   **需求分析與規劃**: 明確專案目標、功能範圍和預期用戶。即使是簡單的筆記或使用者故事，也能幫助聚焦開發方向。
-   **版本控制 (Version Control)**: 使用 Git 等工具追蹤程式碼變更，方便回溯、協作和管理不同版本。
-   **程式碼質量與規範**: 遵循一致的編碼風格、命名約定，保持程式碼清晰、易讀。可以利用 Linting 工具自動檢查。
-   **測試**: ~~編寫單元測試、整合測試，確保每個功能模組按預期工作，並在修改後快速驗證~~。
-   **模組化設計**: 將大問題拆分為小模組，每個模組負責單一職責，降低複雜性。
-   **文檔撰寫**: 雖然小型專案可能無需大量文檔，但關鍵的設計決策、API 使用說明和部署指南仍然是必要的。

### 實務
- 請使用者先提供一份需求簡報（PPT），明確描述每一個操作步驟、輸入條件與預期輸出。
- 釐清需求為前處理、模擬、後處理還是混和？這決定安裝模組與開發框架設定。
- 建立「黃金樣本」：整理範例輸入檔、手動完成的正確輸出檔，以及可重複執行的驗收比對規則。
- 決定介面先行還是算法先行，決定採用同步還是異步框架。
- 由於許多公司限於智財保護，無法提供多個樣本，或是樣本資訊已先被修改，因此開發階段無法涵蓋完整的測試。因此，由客戶自行測試變成是實務的作法。
- 以每週節奏進行進度會報，搭配操作介面 Demo，持續蒐集回饋並更新需求優先級，以釐清需求簡報的不足之處。
- 根據操作時間，操作頻率，決定是否需要排程還是同步還是異步。例如前處理用同步，模擬用異步。
- AI 主導的代碼開發及修正，開發者需借助Prompt, Context 及 Harness Engineering 技巧持續引導 AI Agent完成工作
- 提供 Beta 版本讓使用者測試更多實際案例（含邊界條件），並根據回饋快速迭代修正，以補齊黃金樣本尚未覆蓋的情境。
- 正式交付時同步提供：操作手冊、已知限制清單、維護窗口、版本策略與後續更新計畫。
